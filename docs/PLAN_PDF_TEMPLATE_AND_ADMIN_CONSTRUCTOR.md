# План: шаблонное формирование КП и будущий конструктор в админке

Документ задаёт порядок работ по переходу от генерации первого/второго листа через цельный JPG + overlay к **динамическому формированию по шаблону** (по образцу КП Даичи) и закладывает основу для **конструктора макета в админ-панели** в будущем.

Связанные документы: [PDF_GENERATION_PROCESS.md](PDF_GENERATION_PROCESS.md), образец макета — [КП Даичи пример 17.03.pdf](КП%20Даичи%20пример%2017.03.pdf).

---

## 1. Цели

| Цель | Описание |
|------|----------|
| Шаблонное формирование | Второй лист (КП): только логотип картинкой; заголовок, таблицы заказчика и КП, дата, цены, «Итого» — текстом и таблицами ReportLab по фиксированному шаблону. |
| Динамические данные | Номер КП, дата, описание КП, розничная цена, сумма, итоговая сумма — подставляются из расчёта/БД, без ручной подгонки координат в админке. |
| Третий лист (тех. характеристики) | Номер КП текстом справа от логотипа в зоне технических характеристик; остальная логика страницы без принципиальных изменений. |
| Будущий конструктор | Архитектура и данные заложить так, чтобы позже админка могла задавать позиции/размеры элементов, включать/выключать блоки и переопределять тексты без смены логики генерации. |

---

## 2. Текущее состояние (кратко)

- **Первый лист:** рисуется `first_page.jpg` на весь лист, поверх — один текст (номер ТКП, Inter 20 pt). Таблицы и логотип — часть JPG.
- **Второй лист:** рисуется `second_page.jpg` (или загруженный PDF), поверх — тексты (заголовок КП, описание, цена, сумма, блок «Исполнитель»). Номера КП справа от логотипа нет.
- **Админка:** PdfGenerationSettings хранит флаги страниц, режим второй страницы, загруженные PDF и `text_overlay_config` (координаты/размер шрифта по страницам). Отдельные тексты — PdfTextTemplate.

---

## 3. Целевое состояние (после реализации плана)

- **Второй лист (КП, режим «шаблон из кода»):**
  - Один рисунок — логотип (файл из конфига или админки, cp_logo).
  - Остальное — текст и таблицы: заголовок «КОММЕРЧЕСКОЕ ПРЕДЛОЖЕНИЕ №… от ДД.ММ.ГГГГ» (Open Sans 14 pt), таблица реквизитов заказчика (Open Sans 8 pt), дата и «Цены и условия оплаты» (8 pt), таблица позиций КП с шапкой по центру (8 pt), описание по левому краю по ширине и по центру по высоте (8 pt), строка «Итого, с НДС: …» без наложения на другие элементы.
- **Третий лист (тех. характеристики):** добавлен вывод номера КП текстом справа от логотипа (tech_specs_logo) в начале блока технических характеристик.
- **Подготовка к конструктору:** все координаты, размеры шрифтов и переключаемые блоки выносятся в конфиг/словарь, который в будущем может заполняться из админки (конструктор макета).

### Фаза 4. Будущий конструктор в админ-панели (задел)

Описывается только задел; реализация — отдельный этап.

- [ ] **4.1** Модель или JSON-поле в PdfGenerationSettings для хранения «макета страницы» (второй лист КП, при необходимости третий — тех. характеристики): список блоков с типом (text, table, image), позицией (x, y, width, height), шрифтом, размером, выравниванием, ключом данных. Сохранение в формате, совместимом с п. 4.1.
- [ ] **4.3** При генерации PDF: если в настройках задан макет из конструктора — использовать его; иначе — макет по умолчанию из кода (текущий шаблон). Это позволит не ломать текущее поведение и постепенно переходить на конструктор.

## 5. Зависимости и порядок

- Фаза 0 — перед Фазой 1 (конфиг и структура данных нужны для кода).
- Фаза 1 выполняется до или параллельно с Фазой 2 (независимы по коду, общий конфиг).
- Фаза 3 желательна после 1 и 2, чтобы унифицировать уже работающий шаблон.
- Фаза 4 опирается на формат макета из Фазы 3 и может делаться отдельным спринтом.

---

## 6. Критерии готовности

- Второй лист (КП) генерируется без цельного JPG: только логотип (cp_logo) картинкой, остальное — текст и таблицы с шрифтами Open Sans 14 pt (заголовок) и 8 pt (таблицы, дата, цены, описание, «Итого»); шапка таблицы КП — по центру; описание — по центру по высоте, по левому краю по ширине; строка «Итого» не накладывается на другие элементы.
- Третий лист (тех. характеристики) содержит номер КП текстом справа от логотипа (tech_specs_logo) в зоне технических характеристик.
- Все координаты и размеры второго и третьего листа заданы в конфиге или в явной структуре макета, пригодной для последующей подмены из админки (конструктор).

После выполнения плана обновить [PDF_GENERATION_PROCESS.md](PDF_GENERATION_PROCESS.md) под новую схему генерации второго и третьего листов.
