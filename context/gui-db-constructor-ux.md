# Спецификация UX конструктора БД (`ext`)

Черновик для согласования; стек: React, React Flow (см. [AdminDbConstructor.tsx](../frontend/client/src/components/admin/AdminDbConstructor.tsx)).

## Режимы (tabs / stepper)

1. **Проекты и чертёж** — выбор `ExtSchemaProject`, синхронизация с каталогом, слои графа.
2. **Схема ext (DDL)** — только операции DDL; кнопки порта из Django и apply вынести в мастера.
3. **Данные** — deep link на вкладку «Данные БД» с преселектом таблицы из выбранного узла графа.

## Опасные операции

- Порт из Django, «Применить к PostgreSQL», DROP — мастер: краткое резюме, чекбокс подтверждения, опционально ввод имени таблицы для DROP.

## Превью SQL

Перед apply — запрос списка операций или сгенерированного DDL diff; пользователь видит текст до выполнения.

## Граф

- По умолчанию слой **только ext**; «Показать legacy» — вторичное действие.
- Инспектор узла: колонки, типы, кнопки «Добавить колонку» / FK с выбором целевой таблицы из `ext_catalog`.

## Ошибки и обратная связь

- Единый toast; блок «Последняя операция» с `created/skipped/errors` от API порта.

## Рефакторинг кода

Вынести панели в `components/admin/db-constructor/` (проект, граф, инспектор, диалоги DDL) — см. план миграции.
